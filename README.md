# NixOS Remote Dev Environment Boilerplate

This is a reusable NixOS + Home Manager boilerplate for a remote development box.

Included:

- zsh
- zellij
- eza
- zoxide
- uv
- Docker
- starship prompt
- Rust / rustup / rust-analyzer
- direnv + nix-direnv
- modular Home Manager config
- example Rust project dev shell

## Directory layout

```text
.
├── flake.nix
├── hosts/
│   └── devbox/
│       ├── configuration.nix
│       └── hardware-configuration.nix
├── users/
│   └── nathan/
│       └── home.nix
├── home/
│   └── modules/
│       ├── cli.nix
│       ├── devtools.nix
│       ├── git.nix
│       ├── starship.nix
│       ├── zellij.nix
│       └── zsh.nix
└── devshells/
    └── rust/
        └── flake.nix
```

## First install

Copy this repo to `/etc/nixos` or clone it somewhere and symlink/copy it into place.

Before rebuilding, replace:

```text
hosts/devbox/hardware-configuration.nix
```

with the real file generated by the target machine:

```bash
sudo nixos-generate-config --root /
```

Then rebuild:

```bash
sudo nixos-rebuild switch --flake /etc/nixos#devbox
```

## Where to customize

### Hostname

Edit:

```text
hosts/devbox/configuration.nix
```

Change:

```nix
networking.hostName = "devbox";
```

### Username

Edit:

```text
flake.nix
```

Change:

```nix
username = "nathan";
```

If you rename the user folder, also rename:

```text
users/nathan/home.nix
```

and update the import path in `flake.nix`.

### SSH keys

Edit:

```text
hosts/devbox/configuration.nix
```

Add public keys here:

```nix
openssh.authorizedKeys.keys = [
  "ssh-ed25519 AAAA... your-key-comment"
];
```

Do not put private keys in this repo.

### System packages

Edit:

```text
hosts/devbox/configuration.nix
```

Use this for packages that should exist for the whole system, including before Home Manager loads.

### User packages

Edit:

```text
users/nathan/home.nix
```

or one of:

```text
home/modules/*.nix
```

Use this for user-facing dev tools, shell tools, editor config, prompt config, aliases, etc.

### zsh aliases and login behavior

Edit:

```text
home/modules/zsh.nix
```

The boilerplate currently auto-starts zellij for interactive SSH sessions:

```zsh
if [[ -z "$ZELLIJ" && -n "$SSH_CONNECTION" ]]; then
  zellij attach main -c
fi
```

Comment that out if you prefer a normal shell on login.

### Starship prompt

Edit:

```text
home/modules/starship.nix
```

### Zellij config

Edit:

```text
home/modules/zellij.nix
```

For a full baked-in Zellij config, create:

```text
dotfiles/zellij/config.kdl
```

then uncomment the `xdg.configFile` example.

### Git identity

Edit:

```text
home/modules/git.nix
```

### Rust project shells

Copy this into any Rust project:

```text
devshells/rust/flake.nix
```

Then run:

```bash
direnv allow
```

## Baking in your own dotfiles

Recommended pattern:

```text
dotfiles/
├── nvim/
├── zellij/
│   └── config.kdl
└── git/
```

Then expose files with Home Manager:

```nix
xdg.configFile."zellij/config.kdl".source = ../../dotfiles/zellij/config.kdl;
xdg.configFile."nvim".source = ../../dotfiles/nvim;
```

For text generated directly from Nix:

```nix
xdg.configFile."some-tool/config.toml".text = ''
  setting = "value"
'';
```

## Notes on secrets

Do not bake secrets directly into public or shared Nix files.

For secrets, consider:

- `sops-nix`
- `agenix`
- environment variables injected at runtime
- private files outside the repo

## pi.dev note

The requested `pi.dev` tool/package needs clarification. Once you know the exact CLI/package name, add it to:

```text
home/modules/devtools.nix
```

or package it with an overlay if it is not in nixpkgs.
